Restatify Booking API + WordPress Plugin

Schritt-fuer-Schritt Installation fuer Ubuntu 24.04

Stand: 2026-04-05

Zielarchitektur: WordPress-Server und API-Server getrennt, WireGuard-VPN zwischen beiden, HTTPS ueber Caddy auf dem API-Server, Plugin nutzt API-Basis-URL mit HTTPS.

1. Prerequisites (API-Server, Ubuntu 24.04)

  1. DNS fuer booking-api.deine-domain.tld zeigt auf oeffentliche API-Server-IP.
  2. Ports offen: 22/tcp, 80/tcp, 443/tcp, 51820/udp.
  3. Google Service Account JSON vorhanden (falls Google Sync/Write genutzt wird).
  4. Starker API-Key vorbereitet.
sudo apt update && sudo apt upgrade -y
sudo apt install -y ufw fail2ban unattended-upgrades curl jq git wireguard
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 51820/udp
sudo ufw enable

2. WireGuard VPN zwischen WP- und API-Server

Auf beiden Servern Schluessel erzeugen

umask 077
wg genkey | sudo tee /etc/wireguard/privatekey | wg pubkey | sudo tee /etc/wireguard/publickey

Beispiel API-Server: /etc/wireguard/wg0.conf

[Interface]
Address = 10.44.0.1/24
ListenPort = 51820
PrivateKey = API_SERVER_PRIVATE_KEY

[Peer]
PublicKey = WP_SERVER_PUBLIC_KEY
AllowedIPs = 10.44.0.2/32
PersistentKeepalive = 25

Beispiel WP-Server: /etc/wireguard/wg0.conf

[Interface]
Address = 10.44.0.2/24
PrivateKey = WP_SERVER_PRIVATE_KEY

[Peer]
PublicKey = API_SERVER_PUBLIC_KEY
Endpoint = API_PUBLIC_IP:51820
AllowedIPs = 10.44.0.1/32
PersistentKeepalive = 25
sudo systemctl enable --now wg-quick@wg0
sudo wg show

3. Booking API installieren

Repository auf API-Server kopieren und Installer starten:

cd /tmp/restatify-booking-api
bash install.sh

Der Installer setzt:

  • Systemuser restatify
  • Pfad /opt/restatify-booking-api
  • Python venv + Requirements
  • systemd Service restatify-booking-api
  • Caddy Site mit Reverse Proxy auf 127.0.0.1:8088

4. PostgreSQL einrichten

sudo -u postgres psql
CREATE USER restatify_booking WITH PASSWORD 'SEHR_STARKES_PASSWORT';
CREATE DATABASE restatify_booking OWNER restatify_booking;
\q

5. API Konfiguration (.env)

Datei: /opt/restatify-booking-api/.env

APP_ENV=production
HOST=127.0.0.1
PORT=8088
API_KEY=SEHR_LANGER_ZUFAELLIGER_KEY
DATABASE_URL=postgresql+psycopg://restatify_booking:SEHR_STARKES_PASSWORT@127.0.0.1:5432/restatify_booking
DEFAULT_TIMEZONE=Europe/Berlin
GOOGLE_CREDENTIALS_JSON={...}
GOOGLE_CALENDAR_IDS=
GOOGLE_WRITE_EVENTS_ENABLED=true
GOOGLE_WRITE_CALENDAR_ID=
sudo systemctl restart restatify-booking-api
sudo systemctl status restatify-booking-api --no-pager

6. HTTPS Reverse Proxy mit Caddy

Datei: /etc/caddy/sites-available/restatify-booking-api.caddy

booking-api.deine-domain.tld {
    encode zstd gzip
    reverse_proxy 127.0.0.1:8088
}
sudo systemctl reload caddy
curl -I https://booking-api.deine-domain.tld/health

7. WordPress Plugin installieren

  1. Plugin nach wp-content/plugins/wp-restatify-booking-assistant hochladen.
  2. Im WP-Admin aktivieren.
  3. Einstellungen unter Einstellungen > Booking Assistant setzen:
  • API Basis URL: https://booking-api.deine-domain.tld
  • API Key: derselbe wie in der API .env
  • Zeitzone, Dauer, Suchfenster und Sync-Optionen konfigurieren
  1. Shortcode auf Zielseite platzieren: [restatify_booking_popup]

8. End-to-End Tests

curl https://booking-api.deine-domain.tld/health

Dann im WordPress-Frontend Buchung testen und pruefen:

  • Slot-Suche liefert Ergebnisse
  • Reservierung wird bestaetigt
  • Google-Kalendereintrag wird erstellt (falls aktiviert)
  • Autoresponder-Mail kommt an

9. Security Baseline nach Go-Live

  1. Default API Key sofort ersetzen.
  2. Keine Secrets in Git versionieren.
  3. Backups fuer DB und Konfiguration einplanen.
  4. Fail2ban und Logs regelmaessig monitoren.
  5. Secret-Rotation als naechster Hardening-Schritt umsetzen (ohne Downtime).
